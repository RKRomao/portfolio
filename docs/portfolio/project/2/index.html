NeoLAAC

11/06/2026
microservices docker kubernetes api-gateway devsecops database-per-service python fastapi mariadb rest-api jwt mobile-first ui-ux nginx

O NeoLAAC (Linha de Apoio ao Caloiro) é uma plataforma digital desenhada por e para estudantes da Universidade da Beira Interior (UBI). O seu principal objetivo é atuar como um "mentor digital", simplificando a transição dos novos caloiros ao centralizar recursos académicos, logísticos e sociais para reduzir a ansiedade e a desorientação.

Tecnologias:

Programação & Scripting Python JavaScript (ES6+) HTML5 CSS3 SQL Backend & Microsserviços FastAPI (Python) Node.js Express.js (Node.js) Uvicorn (Servidor ASGI) SQLAlchemy (ORM Python) Bases de Dados MariaDB / MySQL Frontend & Bibliotecas Leaflet.js (Mapas interativos) Leaflet Routing Machine (Rotas e direções) Swiper.js (slides) Font Awesome 6 (Ícones) Google Fonts (Outfit) DevOps & Infraestrutura Docker Docker Compose Kubernetes Nginx (Servidor web e proxy reverso) Cloudflare Tunnel (cloudflared) Ferramentas & Testes Playwright (Testes end-to-end) Pytest / Unittest (Testes unitários em Python) Git

Funcionalidades:

Mapas Interativos do Campus (Localização de blocos e salas) Rotas Pedonais (Navegação GPS até à sala de aula) Informação de Transportes Públicos (Horários e rotas da Covilhã) Chatbot / Mentor Digital (Assistente 24/7 para FAQs) Escalamento Inteligente para Humanos (Transferência de chat para o Staff em caso de emergência ou dúvida complexa) Botão de Pânico / Emergência (Alertas prioritários) Sistema de Tickets de Suporte (Para gestão de incidentes) Sincronização de Horários (Aulas, salas e disciplinas) Calendário Académico (Datas de semestres e exames)\r\nFeed Social Integrado (Avisos institucionais e publicações da comunidade) Espaços para Núcleos e Tunas (Gestão autónoma de eventos) Guias Académicos e de Sobrevivência (Praxe, cantinas, alojamento) Diretório de Parceiros Locais (Benefícios e descontos para estudantes) Perfis de Utilizador Dinâmicos (Estudante, Docente, Staff) Interface Mobile-First (Otimizada para smartphones)

Descrição Detalhada:

Visão Geral e Propósito O NeoLAAC é um ecossistema digital completo e centralizado, desenhado especificamente para apoiar a transição e integração de novos estudantes (caloiros) na Universidade da Beira Interior (UBI) na Covilhã. Atuando como um "mentor digital", a plataforma visa mitigar o atrito logístico, a fragmentação de informação e a ansiedade espacial sentida pelos novos estudantes. Une recursos académicos, logísticos e sociais numa única aplicação mobile-first. Arquitetura de Sistemas Do ponto de vista técnico, o NeoLAAC é construído sobre uma arquitetura de microsserviços robusta e escalável. Orchestration: Toda a comunicação externa passa por um API Gateway/Orchestrator (laac-orq) que lida com encaminhamento, segurança e auditoria centralizada. Contentorização: Todos os componentes estão empacotados como imagens Docker, permitindo escalamento horizontal. Persistência de Dados: Utiliza o padrão "Base de Dados por Serviço", empregando bases de dados MariaDB isoladas (ex: uma base de dados principal e outra separada para dados de calendário) para evitar que o tráfego pesado nos horários académicos degrade a performance de outros serviços como o chat em tempo real. Core Módulos e Funcionalidades A plataforma está dividida em vários microsserviços autónomos que lidam com áreas funcionais distintas: 1. Orientação Espacial e Infraestrutura (map-service) Mapa Interativo do Campus: Utiliza dados espaciais (GeoJSON) para fornecer mapas interativos detalhando faculdades, blocos de edifícios, pisos e salas de aula específicas. Rotas Pedonais: Calcula rotas passo a passo a partir da localização GPS atual do utilizador para uma sala de aula específica. Mobilidade Urbana: Cataloga horários atualizados, rotas e preços das redes de transporte público local da Covilhã. 2. Suporte Híbrido e Gestão de Emergências (chatbot-service, chat-service, emergency-service) Mentor Digital (Chatbot): Os alunos podem fazer perguntas em linguagem natural a um assistente de IA para responder a dúvidas comuns de FAQs. Triagem e Escalabilidade Inteligente: O sistema monitoriza os chats em tempo real. Se o bot detetar palavras-chave críticas (ex: "emergência", "ferido") ou não compreender a questão, o incidente é imediata e automaticamente escalado para funcionários humanos (Frontdesk). Funcionalidades de Emergência: Inclui um "botão de pânico" para alertas imediatos prioritários e a capacidade de solicitar chamadas de emergência diretas. Sistema de Tickets: Todos os incidentes são geridos através de um sistema de tickets que permite à equipa acompanhar o estado (aberto, em progresso, resolvido) dos pedidos dos estudantes. 3. Vida Académica e Logística (academic-service, calendar-service) Calendário Académico: Sincroniza o calendário geral da universidade, incluindo datas de início/fim de semestre e períodos de exames. Horários das Aulas: Permite aos estudantes ver as suas turmas inscritas, disciplinas específicas, horários diários e localizações de salas. Relatório de Erros: Os utilizadores podem submeter tickets especificamente para reportar erros de horários ou salas de aula. 4. Comunidade e Envolvimento Social (post-service, events-service, feed-service) Feed Unificado: Exibe notícias institucionais da UBI juntamente com publicações sociais da comunidade. Organizações Estudantis: Fornece espaços digitais autónomos para grupos de estudantes (Núcleos, Tunas) publicarem anúncios, gerirem eventos e partilharem conteúdo. Guias e Benefícios: Aloja guias de sobrevivência académica, regras de Praxe e um diretório de parceiros comerciais locais verificados que oferecem descontos a estudantes. Segurança e Normas de Qualidade O NeoLAAC impõe requisitos não-funcionais rigorosos para garantir um ambiente seguro e fiável: Segurança: Implementa credenciais encriptadas (ex: bcrypt), autenticação baseada em JWT e Controlo de Acesso Baseado em Funções (RBAC) para gerir permissões em diferentes perfis de utilizador (Estudantes, Professores, Funcionários, Administradores). Resiliência: Conta com cópias de segurança diárias automáticas e endpoints de verificação de estado (/health) para que todos os microsserviços reiniciem automaticamente instâncias degradadas. Performance: O API Orchestrator foi concebido para processar pedidos normais em menos de 200 milissegundos, e o chat em tempo real mantém uma latência inferior a 1 segundo.
